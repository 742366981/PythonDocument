# AI 项目配置

本文档是 AI 项目的配置文件，定义 AI 在本项目中的行为约束和执行规范。Claude Code 工具自动加载此文件。

> **说明**：不同 AI 工具可能使用不同配置文件（`CLAUDE.md` 或 `AGENTS.md`），两者内容保持一致。
>
> **重要**：修改规范时，**必须同时更新两个文件**，确保内容一致。详见 `AI操作规范.md`。

---

## 零、速查表（每次任务前必须对照）

### 强制执行项
- [ ] **修改前**：必须先分析影响、询问确认
- [ ] **完成时**：必须立即 commit（禁止多个操作后才 commit）
- [ ] **接口开发**：必须先写 docstring，再写实现
- [ ] **代码注释**：函数/类/复杂逻辑必须有完整注释
- [ ] **临时文件**：必须放在 `temp/` 目录，用完即删

### 禁止项
- ❌ 未经用户确认直接修改任何代码/文档
- ❌ 先写代码后补文档
- ❌ 只 commit 代码不 commit 文档
- ❌ 多个文档重复定义相同内容
- ❌ 在项目目录创建临时文件

---

## 一、强制行为约束（红线）

| 红线行为 | 违规后果 |
|:---------|:---------|
| 未经确认直接修改代码/文档 | 用户有权要求回滚 |
| 先写代码后补文档 | 视为不合格 |
| 多个操作后才 commit | 视为不规范 |
| 只 commit 代码不 commit 文档 | 视为不规范 |
| 发现重复定义未处理 | 视为不合格 |
| 代码注释不完整 | 视为不合格 |
| 临时文件不清理 | 视为不规范 |

## 二、细节记录机制

### 触发条件
- 用户主动告知关键信息时 → 立即记录
- 用户告知的场景/限制/注意事项 → 记录并后续遵守

### 记录位置
`docs/细节记录/`

### 记录格式
```markdown
## 用户告知的关键信息

| 日期 | 内容 | 用途/关联 |
|-----|------|----------|
| 2026-05-09 | 用户告知的XXX | 用于XXX功能开发 |
```

### 开发前检查
- 修改某功能前 → 检查是否有该功能相关的用户告知记录
- 有记录 → 必须遵守记录内容
- 无记录 → 正常开发

## 三、执行Checklist

### Step 1：识别核心规范
- [ ] 项目类型是否已识别（后端/前端/全栈）？
- [ ] 对应的技术规范文件是否已定位？

### Step 2：查找需求
- [ ] `docs/原始需求/` 和 `docs/需求文档/` 是否已检查？

### Step 3：确认设计文档
- [ ] 设计文档是否存在？用户是否已确认技术选型？

### Step 4：代码执行
- [ ] 是否每个接口都先写了 docstring？
- [ ] 是否每个函数/类都有完整注释？
- [ ] 是否按要求 commit？

### Step 5：细节记录
- [ ] 本次开发是否涉及用户告知的关键信息？
- [ ] 如果是，是否已在 `docs/细节记录/` 中记录？

## 四、项目结构

| 类型 | 路径 | 说明 |
|:-----|:-----|:-----|
| 虚拟环境 | `../venv/{项目名}-env` | 项目平级目录 |
| 原始需求 | `docs/原始需求/` | 用户提供的需求材料 |
| 需求文档 | `docs/需求文档/` | 标准化需求 |
| 设计文档 | `docs/设计文档/` | 架构和技术方案 |
| 技术规范 | `docs/技术规范/` | 技术标准规范 |
| API 文档 | `docs/API文档/` | 接口详细设计，**必须包含完整示例** |
| 细节记录 | `docs/细节记录/` | 用户告知的关键信息 |
| 临时文件 | `temp/` | 用完即删 |

### 4.1 目录职责

| 目录 | 职责 | 编写者 |
|:-----|:-----|:-------|
| 原始需求/ | 保存用户提供 | 用户 |
| 需求文档/ | 标准化需求 | AI |
| 设计文档/ | 定义架构和技术选型 | AI |
| 技术规范/ | 定义技术标准 | AI |
| API文档/ | 定义接口细节，**必须包含完整示例** | AI |
| 细节记录/ | 记录重要细节 | AI |

### 4.2 文档内容要求

**API文档/ 目录必须包含：**

| 文件 | 内容要求 | 参考示例 |
|:-----|:---------|:---------|
| swagger_template.md | Flasgger docstring 编写模板（**必须**） | `docs/API文档/swagger_template.md` |
| API文档.md | 完整接口文档（Markdown格式，**必须**） | `docs/API文档/API文档.md` |

> 注：swagger_spec.json 由 `tools/export_docs.py` 从代码 docstring 导出，无需预置

**内容深度：**
- 完整参数说明（含 type、description、example）
- 响应示例（成功/失败场景）
- 分页参数说明
- 导入导出文件格式说明
