## 强制行为约束

以下规则是 AI 操作的**红线**，**每次对话必须遵守**，违反将被视为不合格：

### 1. 修改前必须询问
- **禁止**直接修改任何代码
- 必须先分析影响范围，询问用户确认后再执行

### 2. 每次操作后必须 commit
- 完成一个接口 → 立即 commit
- 完成一个文档 → 立即 commit
- **禁止**一次性做多个操作后再 commit

### 3. 接口必须先写文档
- **禁止**先写代码后补文档
- 每个接口必须有完整的 docstring

### 4. 禁止重复定义
- 同一概念只定义一次，禁止在多处复制相同内容

### 5. 临时文件必须删除
- 调试文件用完即删，**禁止**堆积在项目中

## 执行Checklist（强制）

**每次任务必须按以下步骤执行，并在回复中明确回答自检问题**：

### Step 0：新项目初始化
- [ ] 是否需要初始化？（检查 .gitignore、语言文件、venv脚本是否存在）
- [ ] 若需要，是否已询问用户确认？

### Step 1：识别核心规范
- [ ] 是否已完整阅读Git规范？
- [ ] 是否已识别项目类型和技术规范？
- [ ] **必须回答：上述检查项是否全部完成？**

### Step 2：查找需求
- [ ] 是否已检查 docs/原始需求/ 和 docs/需求文档/？
- [ ] **必须回答：需求状态是否已明确？**

### Step 3：确认设计文档
- [ ] 设计文档是否存在？用户是否已确认技术选型？
- [ ] **必须回答：是否可以进入代码阶段？**

### Step 4：按流程执行
- [ ] 每个阶段完成后是否立即commit？
- [ ] 是否符合技术规范要求？
- [ ] **必须回答：本阶段是否完成？**
