# 开发环境规范

本文档定义项目开发环境的标准初始化流程，确保不同项目、不同语言、不同开发者之间的环境一致性。

---

## 1. 项目初始化流程（强制）

### 1.1 初始化检查清单

每个新项目创建时，必须按顺序执行：

| 步骤 | 操作 | 说明 |
|:-----|:-----|:-----|
| 1 | 创建 .gitignore | 根据项目类型自动识别并创建 |
| 2 | 创建语言特定文件 | 如 requirements.txt 等 |
| 3 | 创建虚拟环境脚本 | 如 venv_windows.bat |
| 4 | 创建环境配置示例 | 如 .env.example |
| 5 | 更新记忆文件 | 记录虚拟环境路径 |

## 3. Python 项目规范（强制）

### 3.1 必须文件

| 文件 | 说明 |
|:-----|:-----|
| requirements.txt | Linux/macOS 依赖 |
| requirements_windows.txt | Windows 依赖 |
| venv_windows.bat | Windows 虚拟环境创建脚本 |

### 3.2 requirements.txt 格式

```
# 依赖包
flask==3.0.0
flask-cors==4.0.0
flask-compress==1.14
flask-migrate==4.0.5
flask-sqlalchemy==3.1.1
pymysql==1.1.0
redis==5.0.1
python-dotenv==1.0.0
pyyaml==6.0.1
```

### 3.3 requirements_windows.txt

通常与 requirements.txt 相同，如需 Windows 特定依赖可单独添加。

### 3.4 venv_windows.bat

```bat
@echo off
REM 创建 Python 虚拟环境

echo 创建虚拟环境...
python -m pip install -U pip
python -m venv ../venv/{项目名}-env

echo 激活虚拟环境...
cd ../venv/{项目名}-env/Scripts
call activate

echo 升级 pip...
python -m pip install -U pip

echo 安装依赖...
python -m pip install -r ../../../{项目名}/requirements_windows.txt

echo.
echo 完成！激活命令：cd ../venv/{项目名}-env/Scripts && activate
pause
```

## 4. 虚拟环境路径管理（强制）

### 4.1 路径规范（强制）

**虚拟环境统一放在项目平级的 `venv/{项目名}-env` 目录**：

```
workspace/
├── venv/
│   └── {项目名}-env/        ← 虚拟环境目录
│       ├── Scripts/          ← Windows
│       └── bin/              ← Unix
└── {项目名}/                ← 项目目录
    ├── apps/
    └── ...
```

**路径格式**：
- Windows：`../venv/{项目名}-env/Scripts/python`
- Unix：`../venv/{项目名}-env/bin/python`

### 4.2 记忆文件更新（强制）

#### 4.2.1 路径记录格式

| 路径类型 | 格式 | 用途 |
|:---------|:-----|:-----|
| 相对路径 | `../venv/{项目名}-env/Scripts/python` | AI 执行命令时切换环境 |
| 绝对路径 | `<实际路径>/python.exe` | 用户查看/配置 IDE |

#### 4.2.2 更新内容

将虚拟环境路径记录到 `docs/细节记录/` 中：

```markdown
## 环境信息

| 日期 | 内容 | 用途/关联 |
|-----|------|----------|
| 2024-01-01 | 虚拟环境：`../venv/{项目名}-env/Scripts/python` | 环境切换 |
```

## 5. 语言特定规范（强制）

### 5.1 Node.js（强制）

**package.json 基本结构**：

```json
{
  "name": "my-project",
  "version": "1.0.0",
  "scripts": {
    "dev": "node src/index.js",
    "start": "node dist/index.js"
  },
  "dependencies": {},
  "devDependencies": {}
}
```

### 5.2 Go（强制）

**go.mod 基本结构**：

```go
module github.com/username/my-project

go 1.21

require (
    // 依赖
)
```

### 5.3 Java（强制）

**pom.xml 基本结构**：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0">
    <modelVersion>4.0.0</modelVersion>
    <groupId>com.company</groupId>
    <artifactId>my-project</artifactId>
    <version>1.0.0</version>
</project>
```

## 6. 执行检查清单（强制）

### 6.1 每次工作前检查（强制）

**开始工作前必须检查以下项目**：

| 检查项 | 操作 | 如果异常 |
|:-------|:-----|:---------|
| 虚拟环境是否存在 | 检查虚拟环境可执行文件是否存在 | 提示用户创建 |
| 依赖是否完整 | 检查依赖列表是否完整 | 提示用户安装依赖 |

**⚠️ Windows 路径命令格式规范**：
> 路径必须用双引号包起来，参数放在引号外面，用空格分隔
> - ✅ `"../venv/xxx/Scripts/python" --version`
> - ❌ `"../venv/xxx/Scripts/python --version"` （参数被包进引号）

**按项目类型执行检查命令**：

| 项目类型 | 检查命令 |
|:---------|:---------|
| Python | `"../venv/{项目名}-env/Scripts/python" --version` |
| Python | `"../venv/{项目名}-env/Scripts/pip" list` |
| Node.js | `node --version` |
| Node.js | `npm list` |
| Go | `go version` |
| Java | `java -version` |
| Java | `mvn --version` 或 `gradle --version` |
