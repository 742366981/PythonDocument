# 代码变更同步规范

本文档定义代码变更时需要同步更新的内容，确保代码和文档的一致性。

---

## 1. 变更类型分类

| 变更类型 | 说明 | 影响范围 |
|:---------|:-----|:---------|
| 模型变更 | 数据库表结构修改 | 模型文件、swag_from、验证逻辑 |
| 逻辑变更 | 接口业务逻辑修改 | swag_from、模型注释 |
| 路径变更 | API路径或路由修改 | 蓝图注册、swag_from |
| 配置变更 | 配置文件修改 | 所有引用该配置的地方 |
| 新增接口 | 新建API接口 | 模型、视图、swag_from、蓝图注册 |
| 删除接口 | 删除API接口 | 视图、swag_from、蓝图注册 |

---

## 2. 模型变更同步

### 2.1 变更类型与同步项

| 变更类型 | 必须同步更新的文件 |
|:---------|:------------------|
| 新增字段 | 模型文件、swag_from 参数定义 |
| 删除字段 | 模型文件、swag_from 参数定义 |
| 字段类型变更 | 模型文件、swag_from 参数定义 |
| 字段注释变更 | 模型文件、swag_from 参数描述 |
| 唯一约束变更 | swag_from 参数定义 |
| 索引变更 | swag_from 参数定义 |

### 2.2 检查清单

- [ ] 字段名是否正确
- [ ] 字段类型是否正确
- [ ] 字段注释是否与业务一致
- [ ] 唯一约束是否正确
- [ ] 索引是否正确
- [ ] 外键是否使用普通 INT（无外键约束）
- [ ] 时间字段默认值是否正确
- [ ] create_user/update_user 是否存在

### 2.3 示例

**场景**：在 `User` 表新增 `email` 字段

```python
# 1. 模型文件
class User(BaseModel):
    email = db.Column(db.String(100), unique=True, index=True, comment='邮箱')

# 2. swag_from 必须同步
@swag_from({
    'parameters': [
        {'name': 'body', 'in': 'body', 'schema': {
            'properties': {
                'email': {'type': 'string', 'example': 'test@example.com', 'description': '邮箱'}
            }
        }}
    ]
})
```

## 3. 逻辑变更同步

### 3.1 变更类型与同步项

| 变更类型 | 必须同步更新的文件 |
|:---------|:------------------|
| 新增参数校验 | swag_from 参数定义 |
| 删除参数校验 | swag_from 参数定义 |
| 参数校验规则变更 | swag_from 参数定义 |
| 权限变更 | swag_from security 定义 |
| 业务规则变更 | swag_from description |

### 3.2 检查清单

- [ ] 参数校验是否完整
- [ ] 必填参数是否有检查
- [ ] 参数类型是否正确转换
- [ ] 权限装饰器是否正确
- [ ] 错误码是否正确返回
- [ ] 创建人/更新人是否正确填充

### 3.3 示例

**场景**：登录接口新增 `captcha_id` 参数

```python
# 1. 逻辑代码
captcha_id = data.get('captcha_id')
if not captcha_id:
    return api_error(400, '请输入验证码ID')

# 2. swag_from 必须同步
@swag_from({
    'parameters': [
        {'name': 'body', 'in': 'body', 'schema': {
            'properties': {
                'captcha_id': {'type': 'string', 'example': 'a1b2c3d4', 'description': '验证码ID'}
            }
        }}
    ]
})
```

## 4. 路径变更同步

### 4.1 变更类型与同步项

| 变更类型 | 必须同步更新的文件 |
|:---------|:------------------|
| 路径重命名 | 蓝图注册、swag_from |
| 路由方法变更 | swag_from HTTP method |
| 路径参数变更 | swag_from 参数定义 |

### 4.2 检查清单

- [ ] 蓝图注册路径是否正确
- [ ] swag_from 路径是否一致
- [ ] 路径参数是否匹配

## 5. 配置变更同步

### 5.1 变更类型与同步项

| 变更类型 | 必须同步更新的文件 |
|:---------|:------------------|
| 新增配置项 | 所有使用该配置的地方 |
| 配置项删除 | 所有引用该配置的地方 |
| 配置项值变更 | 相关业务逻辑 |

### 5.2 检查清单

- [ ] 配置项是否在正确位置定义
- [ ] 所有引用是否已更新
- [ ] 不同环境的配置是否一致

- [ ] 蓝图注册：

### 3. 确认事项

## 7. 审查要点

### 7.1 自查问题

1. 这个变更会影响哪些接口？
2. 影响的接口的 swag_from 是否同步更新？
3. 模型文件的字段定义是否与 swag_from 一致？
4. 参数校验逻辑是否与 swag_from 一致？
5. 创建人/更新人是否正确填充？

### 7.2 禁止行为

- ❌ 只修改模型文件，不更新 swag_from
- ❌ 只修改逻辑，不更新模型注释
- ❌ 只修改 swag_from，不同步模型
- ❌ 只修改代码，不更新文档

---

## 8. 常见遗漏问题

### 8.1 模型变更遗漏

| 问题 | 后果 |
|:-----|:-----|
| 新增字段未在 swag_from 定义 | API 文档不完整 |
| 字段类型变更未同步 | 可能导致类型不匹配 |
| 注释未同步更新 | 文档与代码不一致 |

### 8.2 逻辑变更遗漏

| 问题 | 后果 |
|:-----|:-----|
| 新增校验未在 swag_from 定义 | 文档无法提示必填 |
| 权限变更未同步 | 文档与实际不符 |
| 创建人/更新人遗漏 | 数据追溯问题 |

### 8.3 文档变更遗漏

| 问题 | 后果 |
|:-----|:-----|
| 接口新增未添加 swag_from | API 文档缺失 |
| 路径变更未更新注册 | 接口无法访问 |
