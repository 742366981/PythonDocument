# 测试规范

本文档定义项目的测试规范，适用于所有项目。

---

## 1. 测试目的

代码完成后，通过测试验证：
1. 代码实现是否符合设计文档
2. 功能是否满足需求文档
3. 边界条件和异常处理是否正确

---

## 2. 测试目录结构

```
project/
├── tests/                      # 测试目录
│   ├── __init__.py
│   ├── conftest.py            # pytest 配置和公共 fixtures
│   ├── fixtures/              # 测试数据
│   │   └── *.json
│   ├── test_*.py            # 测试文件
│   └── report.md             # 测试报告
```

## 3. 测试文件编写

### 3.1 conftest.py 基础配置

```python
# -*- coding: utf-8 -*-
"""
pytest配置和公共fixture
"""

import os
import pytest

BASE_DIR = os.path.dirname(os.path.dirname(__file__))


@pytest.fixture(scope='session')
def app():
    """创建测试应用"""
    from apps import create_app
    app = create_app(protect_swagger=False)
    app.config['TESTING'] = True
    return app


@pytest.fixture(scope='session')
def client(app):
    """创建测试客户端"""
    return app.test_client()


@pytest.fixture(scope='function', autouse=True)
def db_session(db, app):
    """每个测试函数独立的数据库会话"""
    with app.app_context():
        yield
```

### 3.2 测试类组织

```python
# -*- coding: utf-8 -*-
"""
认证接口测试
"""

import pytest


class TestAuthLogin:
    """登录测试"""

    def test_login_success(self, client):
        """测试登录成功"""
        response = client.post('/{项目前缀}/auth/login',
                              json={"username": "admin", "password": "xxx"})
        assert response.status_code == 200
        data = response.get_json()
        assert data['code'] == 0
        assert 'token' in data['data']
```

## 7. 测试原则

### 7.1 测试环境原则

| 原则 | 说明 |
|:-----|:-----|
| 独立配置 | Mock 测试使用独立的配置文件（如 `config/mock.ini`），格式与原配置相同，值为 Mock 专属数据；原有三个环境配置（dev/test/prod）不能被修改或覆盖 |
| 环境分离 | Mock 测试环境与生产环境完全隔离，配置相互独立 |
| 数据隔离 | Mock 测试数据仅在测试环境使用，不影响生产数据 |
| 环境配置 | 通过环境变量或配置文件管理不同环境配置 |
| 独立数据库 | Mock 测试使用独立的数据库或数据库实例 |
| 智能配置 | AI 执行测试时，可根据测试需求自动生成合理的 Mock 配置值（如测试数据库、测试账号等） |

**Mock 测试配置文件示例**：
```ini
# config/mock.ini（格式与原配置相同）
[database]
host = 192.168.1.100
name = app_mock

[redis]
host = 192.168.1.101
port = 6379
```

**环境配置示例**：
```python
# conftest.py
import os

TESTING = os.getenv('ENV_TYPE') == 'test'

if TESTING:
    DB_HOST = 'test-db.example.com'
else:
    DB_HOST = 'prod-db.example.com'
```

### 7.2 测试执行原则

| 原则 | 说明 |
|:-----|:-----|
| 独立性 | 每个测试函数独立运行，不依赖其他测试 |
| 可重复 | 测试可以多次运行，结果一致 |
| 自包含 | 测试数据在 fixtures 中管理 |
| 明确断言 | 每个断言都有明确的预期结果 |
