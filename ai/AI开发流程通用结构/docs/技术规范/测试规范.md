# 测试规范

本文档定义项目的测试规范，适用于所有项目。

---

## 1. 测试目的

代码完成后，通过测试验证：
1. 代码实现是否符合设计文档
2. 功能是否满足需求文档
3. 边界条件和异常处理是否正确

---

## 2. 测试目录结构

```
project/
├── tests/                      # 测试目录
│   ├── __init__.py
│   ├── conftest.py            # pytest 配置和公共 fixtures
│   ├── fixtures/              # 测试数据
│   │   └── *.json
│   ├── test_*.py            # 测试文件
│   └── report.md             # 测试报告
```

---

## 3. 测试文件编写

### 3.1 conftest.py 基础配置

```python
# -*- coding: utf-8 -*-
"""
pytest配置和公共fixture
"""

import os
import pytest

BASE_DIR = os.path.dirname(os.path.dirname(__file__))


@pytest.fixture(scope='session')
def app():
    """创建测试应用"""
    from apps import create_app
    app = create_app(protect_swagger=False)
    app.config['TESTING'] = True
    return app


@pytest.fixture(scope='session')
def client(app):
    """创建测试客户端"""
    return app.test_client()


@pytest.fixture(scope='function', autouse=True)
def db_session(db, app):
    """每个测试函数独立的数据库会话"""
    with app.app_context():
        yield
```

### 3.2 测试类组织

```python
# -*- coding: utf-8 -*-
"""
认证接口测试
"""

import pytest


class TestAuthLogin:
    """登录测试"""

    def test_login_success(self, client):
        """测试登录成功"""
        response = client.post('/api/auth/login',
                              json={"username": "admin", "password": "xxx"})
        assert response.status_code == 200
        data = response.get_json()
        assert data['code'] == 0
        assert 'token' in data['data']
```

## 4. 测试用例来源

测试用例应覆盖需求文档中的：

| 需求类型 | 测试点 |
|:---------|:-------|
| 接口列表 | 每个接口是否实现 |
| 参数说明 | 必填参数、选填参数、边界值 |
| 响应格式 | 成功响应、失败响应 |
| 业务逻辑 | 状态流转、条件判断 |

---

## 5. 测试执行

```bash
# 运行所有测试
pytest tests/ -v

# 运行指定文件
pytest tests/test_auth.py -v

# 生成覆盖率报告
pytest tests/ --cov=. --cov-report=html
```

## 6. 测试报告（强制）

测试完成后，必须生成测试报告。

### 6.1 报告位置

```
project/tests/report.md
```

### 6.3 报告要求

| 要求 | 说明 |
|:-----|:-----|
| 需求覆盖 | 每个需求点都必须有对应测试 |
| 接口覆盖 | 每个接口都必须测试 |
| 结果记录 | 必须记录实际测试结果 |
| 问题记录 | 未通过的用例必须详细记录 |

---

## 7. 测试原则

| 原则 | 说明 |
|:-----|:-----|
| 独立性 | 每个测试函数独立运行，不依赖其他测试 |
| 可重复 | 测试可以多次运行，结果一致 |
| 自包含 | 测试数据在 fixtures 中管理 |
| 明确断言 | 每个断言都有明确的预期结果 |
