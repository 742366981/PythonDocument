# AI 开发流程通用结构

本文档定义 AI 辅助开发的标准流程和规范体系，确保开发过程规范化、可追溯、可复用。

---

## 一、规范体系概览

```
AI操作规范.md          ← AI 行为准则，核心入口
CLAUDE.md / AGENTS.md  ← AI 配置文件，开机自动加载
docs/
├── 技术规范/          ← 技术标准规范
│   ├── API规范.md              通用 API 设计规范
│   ├── Flask后端规范.md        Flask 特定规范
│   ├── Vue前端规范.md           Vue 特定规范
│   ├── Git规范.md               Git 使用规范
│   ├── 开发环境规范.md          项目初始化规范
│   ├── 代码同步修改规范.md       变更同步规范
│   └── 爬虫规范.md              爬虫专用规范
├── API文档/           ← 接口详细设计（必须包含完整示例）
│   ├── swagger_template.md      Flasgger 编写模板
│   ├── swagger_spec.json        Swagger JSON 导出
│   └── API文档.md               完整接口文档示例
├── 原始需求/          ← 用户提供的原始需求材料（用户上传）
├── 需求文档/          ← 标准化后的需求文档
├── 设计文档/          ← 项目架构和技术方案
└── 细节记录/          ← 重要细节记录（业务逻辑、踩坑等）
```

---

## 二、目录职责

| 目录 | 职责 | 编写者 | 上传时机 |
|:-----|:-----|:-------|:---------|
| 原始需求/ | 保存用户提供 | 用户 | 任务开始前 |
| 需求文档/ | 标准化需求 | AI | 需求分析后 |
| 设计文档/ | 架构和技术选型 | AI | 设计阶段 |
| 技术规范/ | 技术标准 | AI | 项目初始化时 |
| API文档/ | 接口细节 | AI | 接口开发时 |
| 细节记录/ | 重要细节 | AI | 开发过程中 |

---

## 三、API文档要求

`docs/API文档/` 目录必须包含：

| 文件 | 内容要求 |
|:-----|:---------|
| swagger_template.md | Flasgger docstring 编写模板（**必须**） |
| API文档.md | 完整接口文档（Markdown格式，**必须**） |

> 注：swagger_spec.json 由 `tools/export_docs.py` 从代码 docstring 导出，无需预置

**内容深度要求：**
- 每个接口必须有完整参数说明（含 type、description、example）
- 每个接口必须有响应示例（成功/失败场景）
- 列表接口必须包含分页参数说明
- 导入导出接口必须包含文件格式说明

## 四、快速开始

### 1. 识别项目类型

根据项目特征识别技术栈：

| 特征 | 类型 | 对应规范 |
|:-----|:-----|:---------|
| requirements.txt | Python | Flask后端规范.md |
| package.json | Node.js | Vue前端规范.md |
| 爬虫脚本 | 爬虫 | 爬虫规范.md |

### 2. 执行检查清单

```
- [ ] 检查 .gitignore 是否存在
- [ ] 确认技术规范文件
- [ ] 检查 docs/原始需求/ 和 docs/需求文档/
- [ ] 设计文档是否存在
```

### 3. 开发流程

```
需求分析 → 技术选型 → 架构设计 → 接口开发 → 测试验证
```

## 六、Commit 规范

**每次操作后立即 commit：**

```bash
git add <文件>
git commit -m "<类型>: <简短描述>"
```

**类型标签：**

| 类型 | 适用场景 |
|:-----|:---------|
| feat | 新增功能 |
| fix | 修复 bug |
| refactor | 重构 |
| docs | 文档 |
| config | 配置 |
| test | 测试 |
| chore | 杂项 |
