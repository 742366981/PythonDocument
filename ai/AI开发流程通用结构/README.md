# AI 开发流程通用结构

AI 辅助开发的标准流程和规范体系。

---

## 项目结构

```
AI开发流程通用结构/
├── AI操作规范.md          ← AI 行为准则（核心入口）
├── CLAUDE.md             ← Claude Code 配置
├── AGENTS.md             ← AI 工具配置
├── docs/
│   ├── 技术规范/          ← 技术标准规范
│   ├── API文档/           ← 接口文档示例
│   ├── 原始需求/          ← 用户需求（用户上传）
│   ├── 需求文档/          ← 标准化需求
│   ├── 设计文档/          ← 架构和技术方案
│   ├── 细节记录/          ← 重要细节记录
│   └── .gitkeep          ← 空目录占位
└── tools/                ← 工具脚本
```

---

## 快速开始

1. **识别项目类型**：根据 `requirements.txt` / `package.json` 等文件判断
2. **执行检查清单**：`AI操作规范.md` 第七章
3. **按流程开发**：需求分析 → 技术选型 → 架构设计 → 接口开发

## Commit 规范

```bash
git add <文件>
git commit -m "<类型>: <描述>"
```

| 类型 | 适用场景 |
|:-----|:---------|
| feat | 新增功能 |
| fix | 修复 bug |
| docs | 文档 |
| refactor | 重构 |
| config | 配置 |
| test | 测试 |
| chore | 杂项 |
